## 🎯 About the Project

ThirtySix Studio is a premium portfolio template designed for creative agencies, freelancers, and studios who want to make a lasting impression. Built with performance and aesthetics in mind, it combines React's component-based architecture with Tailwind's utility-first approach and GSAP's powerful animation capabilities.

### ✨ What Makes It Special

- **Performance-First**: Optimized for speed and smooth interactions
- **Mobile-Responsive**: Pixel-perfect design across all devices
- **Animation-Rich**: Fluid, professional animations that enhance user engagement
- **Highly Customizable**: Easy to adapt to your brand and content needs

## 🚀 Features

### Core Functionality
- **🎨 Smooth Animations** - Advanced GSAP-powered transitions and micro-interactions
- **📱 Responsive Design** - Flawless experience across desktop, tablet, and mobile

## 📦 Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:
- **Node.js** (v16.0 or higher) - [Download here](https://nodejs.org/)
- **npm** or **yarn** - Package manager
- **Git** - Version control

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/euii-ii/thirty-six-studio-clone.git
   cd thirtysixstudio
   ```

2. **Install dependencies**
   ```bash
   npm install
   # or
   yarn install
   ```

3. **Start development server**
   ```bash
   npm run dev
   # or
   yarn dev
   ```

4. **Open your browser**
   Navigate to `http://localhost:5173` to see the project in action.

### Additional Scripts

```bash
# Build for production
npm run build

# Preview production build
npm run preview

# Lint code
npm run lint

# Format code
npm run format
```

## 🎨 Customization

### Styling
- Modify `tailwind.config.js` for theme customization
- Custom CSS in `src/styles/`
- Component-specific styles using Tailwind utilities

### Content
- Update content in respective component files
- Modify images in `public/images/`
- Customize animations in component files

### Configuration
- Environment variables in `.env`
- Build settings in `vite.config.js`
- Animation settings in GSAP implementations
